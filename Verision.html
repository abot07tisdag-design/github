Introduktion till GitHub Desktop & GitHub Pages

Uppgiftens mål

Syftet med uppgiften är att lära sig använda GitHub Desktop för versionshantering och att publicera en webbsida online med GitHub Pages.

GitHub Desktop gör det enklare att arbeta med Git och GitHub genom ett mer visuellt och användarvänligt gränssnitt.

Del 1: Skapa GitHub-konto och installera GitHub Desktop

1. Skapa ett GitHub-konto

Börja med att skapa ett gratis konto på GitHub om du inte redan har ett.

2. Installera GitHub Desktop

Ladda ner GitHub Desktop och installera programmet på din dator. När installationen är klar loggar du in med ditt GitHub-konto.

Del 2: Skapa och ladda upp ett projekt

Vad är ett repository?

Ett repository, eller repo, fungerar som en digital mapp där alla filer och ändringar i ett projekt sparas. GitHub gör det möjligt att:

  • Spara projekt och hålla koll på ändringar
  • Samarbeta med andra
  • Återgå till äldre versioner om något går fel

Skapa ett lokalt projekt

Skapa en ny mapp på datorn och lägg till en HTML-fil och en CSS-fil.

Exempel på HTML-kod

<!DOCTYPE html>
<html lang="sv">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Min första webbsida</title>
    <link rel="stylesheet" href="style.css">
</head>

<body>
    <h1>Välkommen till min första webbsida!</h1>
    <p>Det här är en webbsida som jag hostar via GitHub.</p>
</body>
</html>

3. Lägg till projektet i GitHub Desktop

  • Öppna GitHub Desktop och välj Add local repository.
  • Välj projektmappen på datorn.
  • Om ett felmeddelande visas om att mappen inte är ett Git repository, klicka på Create a repository.
  • Fyll i information om projektet och klicka sedan på Publish repository.
  • Se till att alternativet Keep this code private inte är markerat.

4. Commit och synkronisering

En commit fungerar som en sparad version av projektet. När man gör en commit sparas alla ändringar som har gjorts lokalt på datorn.

Exempel på commitmeddelande

Om man lägger till en ny sida som heter “About us” kan commitmeddelandet vara: 

Add an "About us" page

Efter att commiten skapats klickar man på Push origin för att synkronisera ändringarna med GitHub.

5. Publicera sidan med GitHub Pages

  • Gå till repositoryts inställningar på GitHub.
  • Välj Pages under “Code and automation”.
  • Under Source väljer du Deploy from branch.
  • Välj branchen Main och klicka på Save.

Efter några minuter publiceras webbsidan online via GitHub Pages.

Exempel på URL:
https://dittanvandarnamn.github.io/minwebbsida

6. Reflektion och utvärdering

GitHub Desktop gör det enklare att hålla koll på ändringar i ett projekt. Programmet visar tydligt vilka filer som ändrats och gör det enkelt att skapa commits.

Versionshantering är viktigt eftersom man kan återgå till äldre versioner om något går fel. Det gör också samarbeten enklare eftersom flera personer kan arbeta i samma projekt samtidigt.

GitHub Pages är en bra lösning för att publicera webbsidor eftersom det är gratis, enkelt att använda och gör det möjligt att snabbt dela projekt online.
